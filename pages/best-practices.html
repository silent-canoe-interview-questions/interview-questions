SQL Server Best Practices

Query Optimization

Understanding Execution Plans

Execution plans show how SQL Server executes your queries. Understanding them is crucial for optimization.

How to View Execution Plans

-- Enable actual execution plan (Ctrl + M)
-- or use this query
SET STATISTICS XML ON;
GO

SELECT * FROM Products WHERE Price > 100;
GO

SET STATISTICS XML OFF;
GO

Common Plan Operators

Operator Description Performance Impact
Table Scan Reads all rows in a table High cost for large tables
Index Seek Uses index to find specific rows Generally efficient
Hash Match Builds hash table for joins Memory intensive

Index Optimization

-- Check index usage statistics
SELECT 
    OBJECT_NAME(i.object_id) as TableName,
    i.name as IndexName,
    ius.user_seeks,
    ius.user_scans,
    ius.user_lookups,
    ius.user_updates
FROM sys.indexes i
LEFT JOIN sys.dm_db_index_usage_stats ius 
    ON i.object_id = ius.object_id 
    AND i.index_id = ius.index_id
WHERE OBJECT_NAME(i.object_id) = 'YourTableName';

Index Design Guidelines

  • Create indexes on frequently used WHERE clause columns
  • Include commonly selected columns in covering indexes
  • Consider the balance between read and write operations
  • Regularly monitor and maintain indexes

Security Best Practices

Authentication and Authorization

Implementing Least Privilege Access

-- Create a role for read-only access
CREATE ROLE ReadOnlyRole;
GO

-- Grant SELECT permissions to the role
GRANT SELECT ON Schema::dbo TO ReadOnlyRole;
GO

-- Create a user and add to role
CREATE USER ReadOnlyUser WITHOUT LOGIN;
GO
ALTER ROLE ReadOnlyRole ADD MEMBER ReadOnlyUser;
GO

Security Checklist

  • Use Windows Authentication when possible
  • Implement role-based access control
  • Regularly audit login attempts and permissions
  • Encrypt sensitive data

Preventing SQL Injection

Bad Practice (Vulnerable to SQL Injection)

-- DON'T DO THIS
DECLARE @sql NVARCHAR(MAX);
SET @sql = 'SELECT * FROM Users WHERE Username = ''' + @UserInput + '''';
EXEC(@sql);

Good Practice (Parameterized Queries)

-- DO THIS
DECLARE @sql NVARCHAR(MAX);
DECLARE @params NVARCHAR(MAX);

SET @sql = N'SELECT * FROM Users WHERE Username = @Username';
SET @params = N'@Username NVARCHAR(50)';

EXEC sp_executesql @sql, @params, @Username = @UserInput;

SQL Coding Standards

Naming Conventions

Object Type Convention Example
Tables Plural nouns, PascalCase Customers, OrderDetails
Columns PascalCase, descriptive FirstName, LastOrderDate
Stored Procedures sp_[Action][Entity] sp_GetCustomerOrders
Indexes IX_[Table]_[Columns] IX_Orders_CustomerID

Code Formatting

Good Formatting Example

SELECT 
    c.CustomerID,
    c.FirstName,
    c.LastName,
    COUNT(o.OrderID) as TotalOrders,
    SUM(od.Quantity * od.UnitPrice) as TotalSpent
FROM 
    Customers c
    INNER JOIN Orders o 
        ON c.CustomerID = o.CustomerID
    INNER JOIN OrderDetails od 
        ON o.OrderID = od.OrderID
WHERE 
    o.OrderDate >= DATEADD(YEAR, -1, GETDATE())
    AND c.Status = 'Active'
GROUP BY 
    c.CustomerID,
    c.FirstName,
    c.LastName
HAVING 
    COUNT(o.OrderID) > 10
ORDER BY 
    TotalSpent DESC;

Performance Optimization

Common Performance Issues and Solutions

Problem:

-- Bad practice
SELECT * FROM Orders 
WHERE YEAR(OrderDate) = 2025;

Solution:

-- Good practice
SELECT * FROM Orders 
WHERE OrderDate >= '2025-01-01' 
AND OrderDate < '2026-01-01';

Problem:

-- Bad practice (cartesian product)
SELECT * FROM Orders, OrderDetails;

Solution:

-- Good practice
SELECT * FROM Orders o
INNER JOIN OrderDetails od 
    ON o.OrderID = od.OrderID;

Database Maintenance

Regular Maintenance Tasks

Index Maintenance

-- Rebuild all indexes
ALTER INDEX ALL ON TableName REBUILD;

-- Update statistics
UPDATE STATISTICS TableName WITH FULLSCAN;

Backup Strategy

-- Full backup
BACKUP DATABASE YourDatabase 
TO DISK = 'C:\Backup\YourDatabase.bak'
WITH INIT;

-- Transaction log backup
BACKUP LOG YourDatabase 
TO DISK = 'C:\Backup\YourDatabase_Log.trn';
Maintenance Checklist
  • Schedule regular backups
  • Monitor database growth
  • Check for index fragmentation
  • Review and update statistics
  • Monitor transaction log size
